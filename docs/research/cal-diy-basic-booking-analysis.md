# Анализ cal.diy для базового booking MVP

Источник исследования: репозиторий `calcom/cal.diy`, commit `032962f5775d64959cce9fd2708c27f0b1b0cabe`.
Использованы только first-party материалы из этого репозитория: исходный код, Prisma-схема, README, OpenAPI JSON и файлы `agents/skills/calcom-api/references/*`.

## Короткий вывод

Для базового сервиса постановки и приема встреч достаточно шести вертикальных блоков: пользователь/профиль, тип встречи, расписание доступности, расчет публичных слотов, создание бронирования, действия над бронированием. В `cal.diy` эти блоки явно представлены как `User`, `EventType`, `Schedule`/`Availability`, slots router, `Booking`/`Attendee` и операции confirm/decline/cancel. Источники: [`EventType`](https://github.com/calcom/cal.diy/blob/032962f5775d64959cce9fd2708c27f0b1b0cabe/packages/prisma/schema.prisma#L156-L250), [`Schedule` и `Availability`](https://github.com/calcom/cal.diy/blob/032962f5775d64959cce9fd2708c27f0b1b0cabe/packages/prisma/schema.prisma#L945-L976), [`Booking` и `Attendee`](https://github.com/calcom/cal.diy/blob/032962f5775d64959cce9fd2708c27f0b1b0cabe/packages/prisma/schema.prisma#L826-L930), [`slots` router](https://github.com/calcom/cal.diy/blob/032962f5775d64959cce9fd2708c27f0b1b0cabe/packages/trpc/server/routers/viewer/slots/_router.tsx#L16-L55).

Главная архитектурная идея для MVP: публичный экран слотов может быть быстрым read-flow, но создание booking обязано заново проверять доступность на сервере перед записью. В `cal.diy` быстрый `isAvailable` прямо помечен как быстрый чек без запроса календарей и bookings, а создание booking отдельно вызывает серверную проверку доступных пользователей/конфликтов. Источники: [`isAvailable.handler.ts`](https://github.com/calcom/cal.diy/blob/032962f5775d64959cce9fd2708c27f0b1b0cabe/packages/trpc/server/routers/viewer/slots/isAvailable.handler.ts#L19-L24), [`RegularBookingService.ts`](https://github.com/calcom/cal.diy/blob/032962f5775d64959cce9fd2708c27f0b1b0cabe/packages/features/bookings/lib/service/RegularBookingService.ts#L862-L986), [`ensureAvailableUsers.ts`](https://github.com/calcom/cal.diy/blob/032962f5775d64959cce9fd2708c27f0b1b0cabe/packages/features/bookings/lib/handleNewBooking/ensureAvailableUsers.ts#L94-L114).

Интеграции календарей, видеоконференций, webhooks, email/SMS и slot reservation в `cal.diy` видны в коде, но для нашего базового MVP их лучше отделить от ядра. Их можно оставить как расширения поверх доменных событий и внешних ссылок, потому что сама запись booking уже живет отдельно от `BookingReference` и внешних календарей. Источники: [`BookingReference`](https://github.com/calcom/cal.diy/blob/032962f5775d64959cce9fd2708c27f0b1b0cabe/packages/prisma/schema.prisma#L801-L824), [`handleConfirmation.ts`](https://github.com/calcom/cal.diy/blob/032962f5775d64959cce9fd2708c27f0b1b0cabe/packages/features/bookings/lib/handleConfirmation.ts#L75-L115), [`handleCancelBooking.ts`](https://github.com/calcom/cal.diy/blob/032962f5775d64959cce9fd2708c27f0b1b0cabe/packages/features/bookings/lib/handleCancelBooking.ts#L444-L524).

## Базовые сущности

### `EventType`

`EventType` в `cal.diy` описывает публично бронируемый тип встречи: `title`, `slug`, `description`, длительность, видимость, владельца/хостов, расписание, локации, поля формы, период бронирования, buffers, минимальное время до бронирования, slot interval, политику подтверждения и ограничения. Источники: [`schema.prisma`](https://github.com/calcom/cal.diy/blob/032962f5775d64959cce9fd2708c27f0b1b0cabe/packages/prisma/schema.prisma#L156-L250), [`schemas.ts`](https://github.com/calcom/cal.diy/blob/032962f5775d64959cce9fd2708c27f0b1b0cabe/packages/features/eventtypes/lib/schemas.ts#L68-L109).

Создание типа встречи в `cal.diy` принимает минимум пользовательских полей (`title`, `slug`, `description`, `length`, `hidden`, `locations`, `scheduleId` и настройки booking), затем связывает тип встречи с owner/user/schedule. Источники: [`create.handler.ts`](https://github.com/calcom/cal.diy/blob/032962f5775d64959cce9fd2708c27f0b1b0cabe/packages/trpc/server/routers/viewer/eventTypes/heavy/create.handler.ts#L45-L86), [`create.handler.ts`](https://github.com/calcom/cal.diy/blob/032962f5775d64959cce9fd2708c27f0b1b0cabe/packages/trpc/server/routers/viewer/eventTypes/heavy/create.handler.ts#L150-L157).

Обновление типа встречи в `cal.diy` намного шире MVP: оно покрывает schedule, period limits, booking limits, destination calendar, custom inputs, recurring events, users/hosts, booking fields, buffers и Cal Video settings. Это хороший ориентир для будущего роста, но для MVP не нужно тащить весь набор сразу. Источники: [`update.handler.ts`](https://github.com/calcom/cal.diy/blob/032962f5775d64959cce9fd2708c27f0b1b0cabe/packages/trpc/server/routers/viewer/eventTypes/heavy/update.handler.ts#L60-L96), [`update.handler.ts`](https://github.com/calcom/cal.diy/blob/032962f5775d64959cce9fd2708c27f0b1b0cabe/packages/trpc/server/routers/viewer/eventTypes/heavy/update.handler.ts#L212-L342), [`update.handler.ts`](https://github.com/calcom/cal.diy/blob/032962f5775d64959cce9fd2708c27f0b1b0cabe/packages/trpc/server/routers/viewer/eventTypes/heavy/update.handler.ts#L650-L706).

Для нашего проекта минимальная модель `EventType` уже близка к нужному ядру: `title`, `slug`, `durationMinutes`, `scheduleId`, `slotIntervalMinutes`, `minimumBookingNoticeMinutes`, buffers, `bookingWindow`, `confirmationPolicy`, `hidden`, `bookingUrl`. Локальная опора: `spec/models/event-types.tsp`, `backend/app/db/models.py`, `frontend/src/lib/types.ts`.

### `Schedule` и `Availability`

В `cal.diy` расписание отделено от типа встречи: `Schedule` принадлежит пользователю, содержит `timeZone`, набор правил `Availability`, может быть расписанием по умолчанию и использоваться event type. Источники: [`schema.prisma`](https://github.com/calcom/cal.diy/blob/032962f5775d64959cce9fd2708c27f0b1b0cabe/packages/prisma/schema.prisma#L945-L976), [`schedules.md`](https://github.com/calcom/cal.diy/blob/032962f5775d64959cce9fd2708c27f0b1b0cabe/agents/skills/calcom-api/references/schedules.md#L16-L24).

Default availability в `cal.diy` задается как рабочие дни понедельник-пятница с 09:00 до 17:00, а helper умеет конвертировать UI-расписание в строки `Availability`. Источники: [`availability.ts`](https://github.com/calcom/cal.diy/blob/032962f5775d64959cce9fd2708c27f0b1b0cabe/packages/lib/availability.ts#L9-L52).

Создание schedule принимает имя, массив интервалов по дням и optional `eventTypeId`, проверяет владение event type и может сразу привязать созданное расписание к типу встречи. Источники: [`create.schema.ts`](https://github.com/calcom/cal.diy/blob/032962f5775d64959cce9fd2708c27f0b1b0cabe/packages/trpc/server/routers/viewer/availability/schedule/create.schema.ts#L3-L22), [`create.handler.ts`](https://github.com/calcom/cal.diy/blob/032962f5775d64959cce9fd2708c27f0b1b0cabe/packages/trpc/server/routers/viewer/availability/schedule/create.handler.ts#L19-L76).

Обновление schedule в `cal.diy` пересоздает availability rules и date overrides через сервисный слой, отдельно проверяя владение расписанием. Это полезный паттерн для нашего backend application layer: не обновлять разрозненные интервалы прямо из route handler. Источники: [`ScheduleService.ts`](https://github.com/calcom/cal.diy/blob/032962f5775d64959cce9fd2708c27f0b1b0cabe/packages/features/schedules/services/ScheduleService.ts#L48-L91), [`ScheduleService.ts`](https://github.com/calcom/cal.diy/blob/032962f5775d64959cce9fd2708c27f0b1b0cabe/packages/features/schedules/services/ScheduleService.ts#L114-L152).

Для нашего проекта стоит сохранить разделение `Schedule` и `EventType`: один schedule можно переиспользовать в нескольких event types, а overrides должны влиять на все слоты, рассчитанные по этому schedule. Локальная опора: `spec/models/schedules.tsp`, `backend/app/db/models.py`, `frontend/src/features/availability/AvailabilityEditor.tsx`.

### `Slot`

В `cal.diy` слоты не являются постоянной основной сущностью booking-domain; они рассчитываются по запросу из event type, availability, busy times, текущих bookings, buffers, minimum notice, period limits и reserved slots. Источники: [`slots-availability.md`](https://github.com/calcom/cal.diy/blob/032962f5775d64959cce9fd2708c27f0b1b0cabe/agents/skills/calcom-api/references/slots-availability.md#L194-L204), [`util.ts`](https://github.com/calcom/cal.diy/blob/032962f5775d64959cce9fd2708c27f0b1b0cabe/packages/trpc/server/routers/viewer/slots/util.ts#L728-L829), [`util.ts`](https://github.com/calcom/cal.diy/blob/032962f5775d64959cce9fd2708c27f0b1b0cabe/packages/trpc/server/routers/viewer/slots/util.ts#L1091-L1170).

Public slots API в `cal.diy` принимает окно времени, timezone, event type id или пару username+slug, duration и параметры reschedule/team, после чего возвращает сгруппированные доступные времена. Источники: [`types.ts`](https://github.com/calcom/cal.diy/blob/032962f5775d64959cce9fd2708c27f0b1b0cabe/packages/trpc/server/routers/viewer/slots/types.ts#L7-L61), [`useAvailableTimeSlots.ts`](https://github.com/calcom/cal.diy/blob/032962f5775d64959cce9fd2708c27f0b1b0cabe/packages/features/bookings/Booker/hooks/useAvailableTimeSlots.ts#L7-L24).

В `cal.diy` есть короткое удержание выбранного слота через `reserveSlot`: uid хранится в cookie, reservation имеет `releaseAt`, а конфликтующие reservation других пользователей фильтруются из выдачи. Источники: [`reserveSlot.handler.ts`](https://github.com/calcom/cal.diy/blob/032962f5775d64959cce9fd2708c27f0b1b0cabe/packages/trpc/server/routers/viewer/slots/reserveSlot.handler.ts#L24-L103), [`util.ts`](https://github.com/calcom/cal.diy/blob/032962f5775d64959cce9fd2708c27f0b1b0cabe/packages/trpc/server/routers/viewer/slots/util.ts#L1216-L1243), [`slots-availability.md`](https://github.com/calcom/cal.diy/blob/032962f5775d64959cce9fd2708c27f0b1b0cabe/agents/skills/calcom-api/references/slots-availability.md#L90-L123).

Для нашего MVP slot reservation можно отложить, если публичный flow остается коротким: выбор времени и отправка формы происходят в одном пользовательском сценарии, а backend при создании booking повторно валидирует доступность. Если появится длинная многошаговая форма, высокая конкуренция или оплата, reservation нужно добавить как отдельную временную сущность. Источники паттерна: [`useSlots.ts`](https://github.com/calcom/cal.diy/blob/032962f5775d64959cce9fd2708c27f0b1b0cabe/apps/web/modules/bookings/hooks/useSlots.ts#L77-L155), [`slots-availability.md`](https://github.com/calcom/cal.diy/blob/032962f5775d64959cce9fd2708c27f0b1b0cabe/agents/skills/calcom-api/references/slots-availability.md#L213-L245).

### `Booking`, `Attendee`, статусы

`Booking` в `cal.diy` хранит `uid`, owner/user, event type, title, responses, `startTime`, `endTime`, attendees, location, status, destination calendar, причины cancellation/rejection, признаки reschedule и metadata. Источники: [`schema.prisma`](https://github.com/calcom/cal.diy/blob/032962f5775d64959cce9fd2708c27f0b1b0cabe/packages/prisma/schema.prisma#L851-L930).

`Attendee` вынесен отдельно от booking и хранит email, имя, timezone, locale, phone number и связь с booking; для нашего MVP достаточно одного attendee в booking, но структура `cal.diy` показывает, почему multi-attendee/guests лучше не смешивать с основной строкой booking слишком рано. Источники: [`schema.prisma`](https://github.com/calcom/cal.diy/blob/032962f5775d64959cce9fd2708c27f0b1b0cabe/packages/prisma/schema.prisma#L826-L841), [`bookingCreateBodySchema.ts`](https://github.com/calcom/cal.diy/blob/032962f5775d64959cce9fd2708c27f0b1b0cabe/packages/features/bookings/lib/bookingCreateBodySchema.ts#L97-L110).

Статусы `cal.diy`: `CANCELLED`, `ACCEPTED`, `REJECTED`, `PENDING`, `AWAITING_HOST`. Для нашего проекта можно сохранить локальные статусы `pending_host`, `pending_attendee`, `confirmed`, `declined`, `cancelled`, но важно явно описать переходы между ними. Источник: [`BookingStatus`](https://github.com/calcom/cal.diy/blob/032962f5775d64959cce9fd2708c27f0b1b0cabe/packages/prisma/schema.prisma#L843-L849).

В `cal.diy` создание booking выставляет `ACCEPTED`, если подтверждение по умолчанию не требуется, иначе `PENDING`. Это напрямую ложится на нашу `confirmationPolicy`: `automatic` может создавать `confirmed`, `host` может создавать `pending_host`, а `attendee` может создавать `pending_attendee`. Источники: [`RegularBookingService.ts`](https://github.com/calcom/cal.diy/blob/032962f5775d64959cce9fd2708c27f0b1b0cabe/packages/features/bookings/lib/service/RegularBookingService.ts#L668-L725), [`createBooking.ts`](https://github.com/calcom/cal.diy/blob/032962f5775d64959cce9fd2708c27f0b1b0cabe/packages/features/bookings/lib/handleNewBooking/createBooking.ts#L181-L220).

## Пользовательские сценарии

### 1. Создать и настроить тип встречи

Пользователь создает event type с названием, slug, длительностью, описанием, видимостью, локациями и расписанием. `cal.diy` валидирует create input через Zod и создает запись через handler/repository. Источники: [`schemas.ts`](https://github.com/calcom/cal.diy/blob/032962f5775d64959cce9fd2708c27f0b1b0cabe/packages/features/eventtypes/lib/schemas.ts#L68-L109), [`create.handler.ts`](https://github.com/calcom/cal.diy/blob/032962f5775d64959cce9fd2708c27f0b1b0cabe/packages/trpc/server/routers/viewer/eventTypes/heavy/create.handler.ts#L45-L86), [`event-types.md`](https://github.com/calcom/cal.diy/blob/032962f5775d64959cce9fd2708c27f0b1b0cabe/agents/skills/calcom-api/references/event-types.md#L63-L125).

Для MVP настройки типа встречи должны покрывать только то, что влияет на публичную booking-страницу и расчет слотов: `title`, `slug`, `description`, `durationMinutes`, `scheduleId`, `slotIntervalMinutes`, `minimumBookingNoticeMinutes`, buffers, booking window, confirmation policy и `hidden`. Расширенные настройки из `cal.diy` вроде team scheduling, recurring, seats, custom inputs, payments, destination calendar и Cal Video стоит оставить за пределами первого release. Источники расширений: [`EventType`](https://github.com/calcom/cal.diy/blob/032962f5775d64959cce9fd2708c27f0b1b0cabe/packages/prisma/schema.prisma#L156-L250), [`update.handler.ts`](https://github.com/calcom/cal.diy/blob/032962f5775d64959cce9fd2708c27f0b1b0cabe/packages/trpc/server/routers/viewer/eventTypes/heavy/update.handler.ts#L60-L96).

### 2. Настроить availability и date overrides

Пользователь задает недельные интервалы доступности и timezone в schedule; отдельные date overrides меняют availability для конкретной даты. `cal.diy` описывает эти понятия в first-party API reference и реализует обновление через `ScheduleService`. Источники: [`schedules.md`](https://github.com/calcom/cal.diy/blob/032962f5775d64959cce9fd2708c27f0b1b0cabe/agents/skills/calcom-api/references/schedules.md#L16-L24), [`schedules.md`](https://github.com/calcom/cal.diy/blob/032962f5775d64959cce9fd2708c27f0b1b0cabe/agents/skills/calcom-api/references/schedules.md#L66-L120), [`ScheduleService.ts`](https://github.com/calcom/cal.diy/blob/032962f5775d64959cce9fd2708c27f0b1b0cabe/packages/features/schedules/services/ScheduleService.ts#L11-L34).

Для MVP date overrides можно поддержать сразу, если они уже есть в текущем контракте, потому что без исключений пользователь не сможет закрыть отпуск, праздник или разовый занятый день. Источник для значимости overrides: [`ScheduleService.ts`](https://github.com/calcom/cal.diy/blob/032962f5775d64959cce9fd2708c27f0b1b0cabe/packages/features/schedules/services/ScheduleService.ts#L48-L57).

### 3. Открыть публичную страницу пользователя и типа встречи

В `cal.diy` страница `/{user}` загружает public profile и список публичных event types, скрытые типы встреч не показываются, а при единственном публичном типе возможен redirect сразу на страницу типа встречи. Источники: [`[user]/page.tsx`](https://github.com/calcom/cal.diy/blob/032962f5775d64959cce9fd2708c27f0b1b0cabe/apps/web/app/%28booking-page-wrapper%29/%5Buser%5D/page.tsx#L16-L53), [`getServerSideProps.ts`](https://github.com/calcom/cal.diy/blob/032962f5775d64959cce9fd2708c27f0b1b0cabe/apps/web/server/lib/%5Buser%5D/getServerSideProps.ts#L78-L119), [`getServerSideProps.ts`](https://github.com/calcom/cal.diy/blob/032962f5775d64959cce9fd2708c27f0b1b0cabe/apps/web/server/lib/%5Buser%5D/getServerSideProps.ts#L165-L211), [`getEventTypesPublic.ts`](https://github.com/calcom/cal.diy/blob/032962f5775d64959cce9fd2708c27f0b1b0cabe/packages/features/eventtypes/lib/getEventTypesPublic.ts#L12-L65).

Страница `/{user}/{type}` получает публичный event data, проверяет наличие event type, возвращает title/description/duration/locations/booking fields/schedule/confirmation flags и отдает эти данные в Booker. Источники: [`[type]/page.tsx`](https://github.com/calcom/cal.diy/blob/032962f5775d64959cce9fd2708c27f0b1b0cabe/apps/web/app/%28booking-page-wrapper%29/%5Buser%5D/%5Btype%5D/page.tsx#L18-L69), [`getServerSideProps.ts`](https://github.com/calcom/cal.diy/blob/032962f5775d64959cce9fd2708c27f0b1b0cabe/apps/web/server/lib/%5Buser%5D/%5Btype%5D/getServerSideProps.ts#L212-L304), [`getPublicEvent.ts`](https://github.com/calcom/cal.diy/blob/032962f5775d64959cce9fd2708c27f0b1b0cabe/packages/features/eventtypes/lib/getPublicEvent.ts#L70-L187), [`getPublicEvent.ts`](https://github.com/calcom/cal.diy/blob/032962f5775d64959cce9fd2708c27f0b1b0cabe/packages/features/eventtypes/lib/getPublicEvent.ts#L558-L604).

Для нашего MVP публичный экран должен уметь: найти event type по `username + slug` или по share token, показать базовые данные встречи, загрузить slots, принять имя/email/timezone attendee и отправить booking request. Источники для публичного booker flow: [`BookerWebWrapper.tsx`](https://github.com/calcom/cal.diy/blob/032962f5775d64959cce9fd2708c27f0b1b0cabe/apps/web/modules/bookings/components/BookerWebWrapper.tsx#L35-L87), [`BookerWebWrapper.tsx`](https://github.com/calcom/cal.diy/blob/032962f5775d64959cce9fd2708c27f0b1b0cabe/apps/web/modules/bookings/components/BookerWebWrapper.tsx#L114-L160), [`BookerEvent`](https://github.com/calcom/cal.diy/blob/032962f5775d64959cce9fd2708c27f0b1b0cabe/packages/features/bookings/types.ts#L34-L72).

### 4. Рассчитать и выбрать слот

Frontend `cal.diy` строит вход для slots endpoint из selected date/month, timezone, username/event slug или eventTypeId, duration и reschedule/team params. Источники: [`useSchedule.ts`](https://github.com/calcom/cal.diy/blob/032962f5775d64959cce9fd2708c27f0b1b0cabe/apps/web/modules/schedules/hooks/useSchedule.ts#L70-L114), [`useSchedule.ts`](https://github.com/calcom/cal.diy/blob/032962f5775d64959cce9fd2708c27f0b1b0cabe/apps/web/modules/schedules/hooks/useSchedule.ts#L122-L154).

Серверный расчет слотов в `cal.diy` учитывает minimum booking notice, date ranges, текущие bookings, out-of-office, busy times, host availability, reserved slots и period limits. ### 6. Подтвердить или отклонить booking

В `cal.diy` confirm/decline принимает `bookingId`, `confirmed`, optional `recurringEventId`, `reason` и флаги email/platform. Handler проверяет доступ пользователя к booking, запрещает повторное подтверждение уже accepted booking и дальше либо вызывает confirmation flow, либо переводит booking в `REJECTED` с причиной. Confirmation flow в `cal.diy` может создать внешнее calendar/video событие, отправить email/SMS и обновить статус booking на `ACCEPTED`. ### 7. Отменить booking

В `cal.diy` cancel endpoint работает по `uid`, а не по integer id, потому что id легче угадать; запрос проходит CSRF validation, session/rate-limit и затем вызывает `handleCancelBooking`. Источники: [`route.ts`](https://github.com/calcom/cal.diy/blob/032962f5775d64959cce9fd2708c27f0b1b0cabe/apps/web/app/api/cancel/route.ts#L16-L55), [`zod-utils.ts`](https://github.com/calcom/cal.diy/blob/032962f5775d64959cce9fd2708c27f0b1b0cabe/packages/prisma/zod-utils.ts#L321-L356).

Cancel handler в `cal.diy` запрещает отмену уже отмененного booking, проверяет пользователя, учитывает настройку `disableCancelling`, может требовать reason, блокирует отмену прошлой встречи, ставит `CANCELLED`, сохраняет reason/cancelledBy, удаляет внешние references и отправляет уведомления. Для MVP обязательное ядро: запретить повторную отмену, не отменять прошлые встречи без явного решения продукта, сохранять reason и переводить только активные статусы в `cancelled`. Источники: [`handleCancelBooking.ts`](https://github.com/calcom/cal.diy/blob/032962f5775d64959cce9fd2708c27f0b1b0cabe/packages/features/bookings/lib/handleCancelBooking.ts#L81-L173), [`handleCancelBooking.ts`](https://github.com/calcom/cal.diy/blob/032962f5775d64959cce9fd2708c27f0b1b0cabe/packages/features/bookings/lib/handleCancelBooking.ts#L357-L405), [`handleCancelBooking.ts`](https://github.com/calcom/cal.diy/blob/032962f5775d64959cce9fd2708c27f0b1b0cabe/packages/features/bookings/lib/handleCancelBooking.ts#L444-L524).
